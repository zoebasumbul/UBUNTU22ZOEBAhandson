SOP: Configure sudo Access in WSL (Ubuntu)
Document Purpose

This SOP describes the procedure to configure sudo access for a user in Windows Subsystem for Linux (WSL) running Ubuntu.

Scope

Applicable to:

Windows systems with WSL installed

Ubuntu distributions running under WSL

Users requiring administrative (sudo) privileges inside WSL

Prerequisites

WSL is installed and functional

User has root access inside WSL
(If root access is unavailable, the steps must be performed by an administrator)

Procedure
Step 1: Identify Installed WSL Distributions

Open Command Prompt and run:

wsl -l


Example output:

Windows Subsystem for Linux Distributions:
Ubuntu-18.04 (Default)
Ubuntu-20.04

Step 2: Set Ubuntu-20.04 as Default Distribution (Recommended)
wsl --set-default Ubuntu-20.04


Verify:

wsl -l


Expected output:

Windows Subsystem for Linux Distributions:
Ubuntu-20.04 (Default)
Ubuntu-18.04

Step 3: Login to Ubuntu as Root
wsl -d Ubuntu-20.04 -u root


Successful login prompt:

root@<hostname>:#

Step 4: Set / Reset User Password

Execute the following command inside the root shell:

passwd zoeba


Enter the new password

Re-enter to confirm
(Password characters will not be displayed)

Step 5: Grant sudo Privileges
usermod -aG sudo zoeba

Step 6: Exit Root Session
exit

  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
